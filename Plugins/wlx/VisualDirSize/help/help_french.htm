1) Description

  • VisualDirSize fournit une vue d'ensemble graphique de tous les fichiers¦dossiers contenus dans un répertoire sélectionné.
  • L'on peut visionner un graphique en plaçant le curseur sur n'importe quel répertoire, puis en appuyant sur <Ctrl+Q>.
  • Le répertoire choisi sera scruté, et les tailles des fichiers¦dossiers qu'il contient s'afficheront sous forme d'histogramme (barres horizontales) dans le panneau de Vue rapide.
  • Un tel histogramme est utile pour afficher les tailles relatives des fichiers¦dossiers.
  • Par exemple : l'on peut rapidement repérer le plus gros fichier¦dossier du répertoire.
  • Vu la façon dont le module fonctionne, il n'est pas recommandé de l'utiliser sur de très gros répertoires, l'attente devenant excessive…

2) Installation

  1. TC v. 5.51
Copiez le fichier <visualdirsize.wlx> dans le dossier des modules du répertoire de Commander.
Sous la section [ListerPlugins], ajoutez ensuite la ligne suivante dans le fichier <wincmd.ini> :
0=C:\totalcmd\plugins\visualdirsize.wlx (changez le chemin pour celui réel en cours )
  1. TC v. 6.5 et plus récentes
Cette version de TC et suivantes ont une fonction automatique d'installation des modules. Ouvrez simplement l'archive du module, et TC vous demandera si vous voulez l'installer.
Avec l'outil gratuit TC Plugman, on peut aussi installer facilement le module avec toutes les versions de TC citées ci-dessus.

3) Emplacement de <VisualDirSize.ini>

  • Par défaut, <VisualDirSize.ini> sera enregistré dans le même répertoire que <Wincmd.ini>.
  • Si cet emplacement ne convient pas, on peut simplement déplacer le fichier vers l'une des destinations suivantes :
  • Dans un sous-répertoire nommé “VisualDirSize” dans l'aborescence du dossier où se trouve <Wincmd.ini>.
  • Le répertoire du module.
  • Celui de Total Commander.

4) Utilisation

  • Placez le curseur sur un répertoire quelconque, puis appuyez sur “Ctrl+Q”.
  • Il est impossible d'exécuter le module par F3, il fonctionne uniquement avec Ctrl+Q.
  • On peut utiliser les touches-flèches ainsi que Début,, Page Suivante et Page Précédente ou <Barre espace>
  • pour de déplacer parmi les barres d'un histogramme.
  • Un double-clic gauche sur une barre de sous-répertoire y donnera accès et scrutera ce sous répertoire,
  • alors qu'un double-clic droit fera remonter au répertoire parent [••].
  • Pour abandonner la scrutation, appuyez assez longtemps sur la touche <Échap>.

6) Types de graphiques

  • VisualDirSize peut afficher les tailles des répertoires en quatre types de graphiques.
  • Un graphique affiche toujours les tailles des sous-répertoires du dossier choisi en cours.
  • Les éléments des graphiques peuvent être triés par taille¦nom et donner la taille en octets &multiples¦pourcentage du total¦nombre de fichiers,
  • selon les choix effectués par l'utlisat¦eur¦rice dans la barre d'outils.
  • Un double-clic gauche sur une barre d'histogramme fait scruter le dossier correspondant.
Histogramme
Voir la capture d'écran
  • Représente le résultat de la scrutation du lecteur¦dossier choisi sous forme d'histogramme horizontal (barres superposées).
Secteurs
Voir la capture d'écran
  • Le résultat de la scrutation du lecteur¦dossier choisi est affiché en cercle par secteurs (plaisamment nommé “camembert”).
  • Dans ce mode, les petits fichiers¦dossiers peuvent tous être rassemblés dans une portion unique nommée “Autres”.
    		•
    Blocs
    Voir la capture d'écran
  • Là, ce sont des blocs colorés qui affichent le résultat de la scrutation du lecteur¦dossier choisi en cours. Si le bloc représente un répertoire, près de lui des blocs représentants tous les fichiers contenus dans ce répertoire et ses sous-répertoires sont affichés.
  • Les dossiers trop petits pour être affichés individuellement seront rassemblés dans un seul bloc.
    		•
    Extensions
    Voir la capture d'écran
  • Le résultat de la scrutation du lecteur¦dossier choisi en cours s'affiche en texte plus histogramme,
  • et indique le nombre de types de fichiers et la taille totale par type (par extension).

    7) Filtrage de la scrutation


    • Cette caractéristique permet de filtrer les fichiers et dossiers scrutés.
    • Avec cette fonction, on peut indiquer précisément quels types de fichiers devront être inclus dans les graphiques.
    • Ainsi, si l'on ne désire obtenir des informations que pour certains types de fichiers, par exemple des fichiers d'images,
    • on peut ici employer le masque : *.jpg *.gif.
    Masques de nom de fichier
     Voir la boîte de dialogue
    1. Dans ce champ, on peut entrer le masque de recherche pour les fichiers que l'on veut trouver.
    2. Les noms contenant des espaces doivent être mis entre guillemets, p. ex.. : "Lettre à Mr. Dupont.doc",
    3. sinon VisualDirSize rechercherait chaque partie du nom, individuellement.
    4. On peut employer plusieurs filtres, séparés par un point-virgule ou un espace.

    Exemples :

    • *.txt; *.exe (*.txt *.exe) cherche tous les fichiers d'extension TXT ou EXE.
    • *.ini peut trouver par exemple win.ini
    • Dupont trouvera "Lettre à Mr. Dupont.doc"
    • Les masques d'exclusion sont placés après une barre verticale “|” (AltGr+6Alpha).

    Exemple :

    • w*.* | *.bak trouve des fichiers qui commencent par w et ne finissent pas par .bak
    • Pour inclure¦exclure certains répertoires de la recherche, leurs noms doivent comporter
    • une barre oblique inversée

    Exemple :

    • windows\ system32\ *.ini ne trouve que les fichiers INI des répertoires “windows\” et “system32”
    • *.ini | windows\ trouve tous les fichiers INI, sauf ceux des répertoires Windows
    • et de leurs sous-répertoires.
    Recherches de TC
    • Utilisez le bouton Rechercher placé à droite du dialogue pour voir les paramètres de recherche
    • enregistrés par Total Commander pour des recherches fréquentes antérieures.
    Utiliser les Expressions régulières Cherche dans les noms de fichiers en employant les Expressions régulières.
    Exclure les fichiers qui correspondent Exclure - plutôt qu'inclure - les fichiers correspondants au masque déclaré.

    Ces options ne seront pas enregistrées en quittant VisualDirSize de façon à éviter des résultats erronés lors de l'utilisation suivante.

    8) Générer un rapport

    • VisualDirSize possède de puissantes possibilités intégrées de création et d'exportation d'un rapport.
    • Les contenus de rapport exportés peuvent souvent être importés et utilisés par des applications externes pour un traitement supplémentaire.
    • Une large gamme de formats d'exportation sont disponibles, comprenant :
    Rapport HTML
    • Permet d'enregistrer le rapport en fichier HTML, que l'on pourra visionner plus tard dans n'importe quel
    • navigateur¦visionneuse HTML, ou dans Lister en texte clair.
    Rapport TXT
  • Enregistre les données collectées dans un fichier texte “séparé par tabulations” pouvant être importé par
  • des applications telles que Microsoft Excel.
    		•
    Image seulement
  • Le graphique de l'application principale peut être exporté directement en fichier d'image JPEG.
    		•

    La fonction d'exportation de rapport est accessible par le bouton “Créer un rapport” de la barre d'outils.

    Fichier d'icônes externe

    • On peut utiliser la variable %COMMANDER_PATH% dans le chemin.
    Couleur du fond
  • Permet de choisir la couleur du fond des graphiques.
    		•
    Couleur du texte
  • Pour établir la couleur du texte des graphiques.
    		•
    Changer de police
  • Pour choisir une police qui peut être normale ou en gras, l'italique n'étant pas assumé ni
  • par VisualDirSize ni par Total Commander.
  • Mais vous pouvez toujours essayer une police seulement italique d'origine, voir ICI s.v.p. !
    • Tri comme WinXP
  • Utilise un tri logique mixte alphabétique et numérique, par exemple :
  • fichier10.txt vient après fichier1.txt.
  • Cette option ne marche que sous Windows XP, dans les autres systèmes, on a le tri habituel.
  • …ce qui n'est pas absolument vrai ! Le tri logique marche très bien sous Win 98 SE
  • dès que l'on a Internet Explorer 6 ou plus récent d'installé, et que l'on a ajouté SortUpper=3
  • dans <wincmd.ini> sous la section [Configuration]. Le module suit ce tri, nous avons vérifié.

    Ne pas en afficher plus de : (barres)
  • Établit combien de barres pourront être affichées (par défaut : 500).

    Un facteur de 2 dessinera deux fois plus d'informations que la fenêtre ne peut en afficher.
